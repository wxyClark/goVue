HTML 定义术语标签 (dt)

<dt> 标签用于在定义列表中定义术语（definition term），与 <dl> 和 <dd> 标签配合使用。

它用于指定要解释或定义的术语，通常与一个或多个 <dd> 标签（定义描述）关联。

基本语法

<dl>
    <dt>术语1</dt>
    <dd>术语1的定义或描述</dd>
    
    <dt>术语2</dt>
    <dd>术语2的定义或描述</dd>
</dl>

使用示例

示例1：基本使用

代码：

<dl>
    <dt>HTML</dt>
    <dd>超文本标记语言，用于创建网页结构。</dd>
    
    <dt>CSS</dt>
    <dd>层叠样式表，用于设计网页外观。</dd>
    
    <dt>JavaScript</dt>
    <dd>脚本语言，用于添加网页交互功能。</dd>
</dl>

效果：

HTML
超文本标记语言，用于创建网页结构。
CSS
层叠样式表，用于设计网页外观。
JavaScript
脚本语言，用于添加网页交互功能。

示例2：带样式的定义术语

代码：

<style>
.custom-dt {
    font-weight: bold;
    color: #2c3e50;
    font-size: 1.1rem;
}
.custom-dd {
    color: #555;
    margin-left: 1.5rem;
    margin-bottom: 1rem;
}
</style>

<dl>
    <dt class="custom-dt">前端开发</dt>
    <dd class="custom-dd">负责构建用户可以直接看到和交互的网页部分。</dd>
    
    <dt class="custom-dt">后端开发</dt>
    <dd class="custom-dd">负责处理服务器端逻辑、数据库交互和API开发。</dd>
    
    <dt class="custom-dt">全栈开发</dt>
    <dd class="custom-dd">同时具备前端和后端开发能力的开发者。</dd>
</dl>

效果：

前端开发
负责构建用户可以直接看到和交互的网页部分。
后端开发
负责处理服务器端逻辑、数据库交互和API开发。
全栈开发
同时具备前端和后端开发能力的开发者。

使用注意事项

  • 语义化意义：<dt> 标签具有明确的语义，用于定义术语，有助于搜索引擎和辅助技术理解内容结构。
  • 嵌套使用：<dt> 只能作为 <dl> 标签的直接子元素。
  • 多个描述：一个 <dt> 可以对应多个 <dd> 元素，表示一个术语有多个定义或描述。
  • 样式控制：默认情况下，浏览器会对 <dt> 应用粗体样式，但可以通过CSS自定义外观。
  • 与其他列表的区别：定义列表（dl）与无序列表（ul）和有序列表（ol）不同，它用于术语-定义的对应关系。

浏览器支持

  • Firefox (所有版本)
  • Safari (所有版本)
  • Edge (所有版本)
  • Opera (所有版本)
